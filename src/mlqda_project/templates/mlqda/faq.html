Why do I need to prepare my files?

The current implementation of the underlying machine learning script cannot identify the unnecessary parts of your text files. Identifying recurring snippets of raw text is important but not something within the current scope of the software. Leaving these unnecessary texts in, could lead to your results being wrong. For example, if you keep the pseudonym for the participant in the text, the script could pick it up as an important word for a topic because it is recurring often. However, in reality, the pseudonym of the participant does not carry any meaningful information regarding th underlying topics of your data. For this reason we ask you to remove any text that is not the text you want to analyse.
The current implementation does not support warnings if your text is not prepared correctly.

What pieces of texts should I remove from my files?

Strictly speaking any text that you do not wish to analyse. Most commonly these include: Speaker names, Interview/focus group questions, Page numbers, Headers, Participant information

How do I remove these unnecessary texts?

There are mult ways to go on about this. Probably the easiest way is to make a copy of your files and just delete the lines that should not be included in the analysis. After deleting these lines, you can upload your new files on either the Sentiment Analyser or the Topic Modelling tab.

Is there file limit?

Yes there is a file limit. Currently, the host server has an allocated memory of 5 GB. As the site is still in a pilot phase this limit should be more than enough for the users. Because the files are automatically deleted this limit only applies to any moment, not an overall constraint. For example, three separate user can upload 1 GB of files each at the same time. However, the server would terminate the uploads when the storage is full until there is enough space to upload your files. Conversely, a user is welcomed to use the site regularly even if the cumulative size of the files are well over 1 GB or even 5 GB. The 1GB limit should be more than enough for an average user as it roughly translates to 10.000 documents or 50.000 pages across documents.

How does the site handle my uploaded files?

All of your uploaded and processed files are deleted from the server as soon as you leave the site. Your files are immediately deleted after processing. After creating the results, all the references and variables using the content is also deleted. The result file is saved until you are on the results page. Upon leaving that page, you have to confirm that you want to leave the page and even your result files are deleted. Please ensure you save your results before exiting the page!

How do I get my results?

After uploading your files, to site should take you a redirect page. Once your results have been analysed you should be automatically redirected to the results page where you can find your results.

How do I download my results?

From the download page you can access the most important results files individually, or you can download every file in zip folder.

How do I interpret my Topic Modelling/Thematic Analysis results?

Your results consists of multiple parts with varying level of depth. You should receive a list of the most important words grouped by topic. These topics are not defined, so you need to deduct the content of the topics based on these words. Furthermore, you should receive a version of your original files where the sentences that contain one of the topic words are highlighted. Finally, there are also a set of other files containing the transformed data like a document-term matrix. You can use these files to run your own script potentially.

How do I interpret my Sentiment Analysis results?

Your results should include sentiment scores. These can be found on the left-hand side of the table. These scores are called 'compound sentiment scores' as they are calculated from negative, neutral and positive sentiment scores. The displayed scores range from -1 to 1, so in essence you can interpret them as percentages. For example a score of +0.25 could be interpreted as 25% positive. When compiling the document, the system calculates the sentiment score for every sentence. These sentence-sentiment scores are then aggregated into a document and a corpora wide sentiment score. This way, you end up with an average sentiment score for each of your uploaded document and with one average sentiment score for all of your documents.

How do I open my results?

The results come in easily accessible formats like .txt or .pdf files.

Are there any caveats?

The scope of this project is limited so far. There a few caveats to keep in mind when using the site.

  • The only dynamic aspect of the topic modelling feature is the number of topics. The appropriate number of topics is calculated based on the best coherence score.
  • All other aspect of LDA topic modelling apart from the number of topics is hard-coded and cannot be changed by the user.
  • The sentiment analyser utilised in the script is a built-in functionality of the NLTK library. This sentiment analyser was trained on annotated Twitter data, so it might have different precision values on texts from other origins.
